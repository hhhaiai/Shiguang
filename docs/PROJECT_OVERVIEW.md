# 项目概述：拾光 (Shiguang)

## 1. 定位与愿景

**目标**：打造一款 100% 本地化、极度注重隐私的 AI 日记与知识碎片捕获工具。它既是“随手记”，也是“可回忆的本地大脑”。

**核心理念**：
- **Local-First**：所有数据与模型优先本地存储与本地推理
- **无感捕获**：语音/文本/截图/OCR 随时录入
- **回忆式检索**：语义检索 + 对话式回忆

## 2. 形态与平台

- **移动端（核心）**：语音极速录入 + 实时流式转写
- **PC 端（常驻）**：全局快捷键唤醒 + 截图/OCR/文本捕捉（规划）
- **Web 端（入口）**：扫码关联 PC，仅作录入入口（规划）

## 3. 关键能力

1. **语音流式录入**：SenseVoice 本地模型 + PCM 流式追加
2. **端侧 AI 处理**：本地摘要、标签、对话检索
3. **日历/闹钟提醒**：AI 抽取待办与日程，端侧触发提醒
4. **可选外部检索**：仅在用户显式开启时，使用开源搜索接口获取公开信息（默认 SearXNG）
5. **语义检索**：ObjectBox HNSW 向量检索
6. **多用户隔离**：本地账号 + PIN 保护，强隔离
7. **编辑与分享**：日记可编辑，生成图片分享

## 4. 技术路线（当前统一方向）

- **核心数据库**：ObjectBox（向量检索优先）
- **语音识别**：SenseVoice (ONNX Runtime)
- **Embedding**：384 维（优先）
- **对话模型**：Qwen2.5-0.5B int4（默认）
- **通信协议**：本地 WebSocket 单连接（PCM）

## 5. 阶段性目标

### Phase 1
- 移动端语音/文本录入
- ObjectBox 本地存储
- 时间轴 UI + 编辑/分享

### Phase 2
- 本地 Embedding + 语义检索
- 端侧摘要/标签
- PC 端快捷键捕获 MVP

### Phase 3
- 对话式检索（RAG）
- 日历/提醒/闹钟融合
- Web 端扫码入口
